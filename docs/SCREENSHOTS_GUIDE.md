# Screenshots to Capture for the Project Report / Submission

Run the app locally (`python app.py`) and capture the following screens.
Save them into a `docs/screenshots/` folder using the suggested filenames —
the project report template references these exact names.

| # | Screenshot | Suggested filename | What to show |
|---|---|---|---|
| 1 | Home Page | `01_home.png` | Full hero section + feature grid |
| 2 | Register Page | `02_register.png` | Empty registration form |
| 3 | Login Page | `03_login.png` | Empty login form |
| 4 | Fraud Detector — Input | `04_detect_input.png` | A fraud SMS pasted, before clicking Analyze |
| 5 | Fraud Detector — Result | `05_detect_result.png` | FRAUD verdict with reasons + tips expanded |
| 6 | Digital Arrest Detector | `06_digital_arrest.png` | Alert banner shown after analyzing a scam transcript |
| 7 | UPI/QR Awareness | `07_upi_qr.png` | UPI ID check result |
| 8 | Report a Scam | `08_report_scam.png` | Filled-in report form |
| 9 | User Dashboard | `09_dashboard.png` | Chart + scan history table with a few entries |
| 10 | Admin Panel | `10_admin_panel.png` | Stats cards + reports table |
| 11 | Mobile View | `11_mobile_responsive.png` | Home page at ~390px width (use browser dev tools) |

## Tips
- Use your browser's device toolbar (F12 → Ctrl+Shift+M) for the mobile screenshot.
- Populate the dashboard with 5–6 varied scans (mix of safe/suspicious/fraud) before capturing, so the chart looks meaningful.
- Keep the browser window clean — close dev tools, bookmarks bar, etc. before capturing (except for the mobile shot).
- PNG format, at least 1280px wide for desktop shots.
